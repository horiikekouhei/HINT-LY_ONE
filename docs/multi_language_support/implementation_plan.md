# 多言語表示および部屋ごとの言語設定の実装計画

ユーザーの画面表示言語を切り替える機能（日本語・英語など）と、部屋（ルーム）作成時にその言語に応じたお題が出題される仕様を実装します。

## 概要

1. **画面表示の多言語化 (i18n)**
   - アプリケーション全体のUIテキスト（ボタン、説明文、メッセージなど）を多言語対応します。
   - 外部ライブラリ（`react-i18next`等）を導入するか、軽量なカスタムContextを使用します（今回はパフォーマンスとシンプルさを重視し、カスタムContextによる実装を提案します）。
   - ロビー画面右上に言語切り替えボタン（JP / EN）を配置し、ユーザーの選択言語を `localStorage` に保存します。

2. **部屋作成時の言語設定**
   - 部屋を作成した際のホストの「選択言語」を、Firebaseの部屋データ (`Room`) に保存します。
   - 部屋の言語は、参加者がどの言語で参加しようとも「その部屋で出題されるお題の言語」として固定されます。

3. **多言語のお題リスト (Topic List)**
   - 既存の `src/data/topics.ts` を拡張し、日本語 (`TOPIC_LIST_JA`) と英語 (`TOPIC_LIST_EN`) の両方のリストを用意します。
   - 部屋の言語設定に応じて、適切なお題リストが選ばれてゲームが進行するようにします。

## 確認事項 (User Review Required)

以下の点について、実装方針のご確認をお願いします。

1. **翻訳の管理方法について**
   今回は外部ライブラリを追加せず、独自に `LanguageContext` と翻訳辞書ファイル (`ja.ts`, `en.ts`) を作成して実装する方針でよろしいでしょうか？（小〜中規模アプリの場合はこの方が動作が軽く、管理もシンプルです。`react-i18next` などの標準ライブラリを使いたい場合はお知らせください）

2. **言語切り替えの動作範囲について**
   部屋に設定された言語（例: 英語）でゲームが進行している際、他の参加者のUI（ボタン等）の言語は「各自が選んでいる言語」のままにし、**「出題されるお題」だけが部屋の言語**（英語）になる仕様でよろしいでしょうか？（それとも、部屋の言語に合わせて全員のUI言語も強制的に切り替える方がよいでしょうか？）

## Proposed Changes

### 1. Types & Data
Firebaseのデータ構造と、お題のリストを多言語対応させます。

#### [MODIFY] `src/types/game.ts`
- `Room` インターフェースに `language: 'ja' | 'en'` プロパティを追加。

#### [MODIFY] `src/data/topics.ts`
- 現在の `TOPIC_LIST` を `TOPIC_LIST_JA` に変更し、新たに英単語のリスト `TOPIC_LIST_EN` を追加。

### 2. State Management
状態管理（ストア）の変更を行います。

#### [MODIFY] `src/store/gameStore.ts`
- `createRoomInFirebase` 関数に `language` 引数を追加し、指定された言語を部屋データに保存。
- 部屋の言語に応じて、使用するお題リスト（`TOPIC_LIST_JA` または `TOPIC_LIST_EN`）を切り替えるよう修正。

### 3. Localization (i18n)
UIテキストを管理・提供する仕組みを新設します。

#### [NEW] `src/i18n/LanguageContext.tsx`
- 現在の言語状態を管理・提供し、翻訳関数 `t('キー')` を提供する Context と Provider を作成。
- 選択言語は `localStorage` に保存。

#### [NEW] `src/i18n/locales/ja.ts` & `en.ts`
- UIテキストの日本語版、英語版の辞書ファイルを作成。

### 4. UI Components
各画面に翻訳関数を適用し、言語切り替えボタンを配置します。

#### [MODIFY] `src/App.tsx` または `src/main.tsx`
- アプリケーション全体を `LanguageProvider` でラップ。

#### [MODIFY] `src/pages/Lobby.tsx`
- 画面右上に言語切り替えトグル（🌐 JP | EN）を追加。
- ハードコードされているUIテキストを `t()` 関数に置き換え。
- 部屋作成時に、現在選択されている言語を `handleCreateRoom` に渡す。

#### [MODIFY] 各画面・コンポーネント
- `WaitingRoom.tsx`, `Game.tsx`, `GameSummary.tsx`, `Phase1Topic.tsx` 〜 `Phase5Result.tsx` 内のテキストを `t()` に置き換え。
