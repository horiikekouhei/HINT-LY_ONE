# 実装計画: 1,000単語への語彙拡張と多言語対応

「Just One」ゲームの再プレイ性と国際対応を強化するため、全サポート言語の語彙を大幅に拡充し、UIと言語設定の統合を行いました。

## 実施目的
- 各言語の語彙を100語程度から1,000語へと拡張し、飽きのこないゲーム体験を提供する。
- 語彙を意味のあるカテゴリで分類し、ランダム選択時の偏りを防ぐ。
- UI言語の切り替えと、作成される部屋の言語設定を同期させる。

## 提案された変更

### 1. 語彙データの大規模拡張 (`src/data/topics.ts`)
- **対応言語**: 日本語、英語、簡体字中国語、繁体字中国語、韓国語、スペイン語、フランス語、ヒンディー語、アラビア語。
- **カテゴリ化**: 以下の9カテゴリに分類し、合計1,000単語/言語を達成。
    - 食事・飲み物 (150)
    - 動物 (150)
    - 乗り物・交通 (100)
    - 建物・場所 (120)
    - 日用品・家電 (150)
    - 衣類・ファッション (100)
    - 職業・役割 (80)
    - スポーツ・趣味 (80)
    - 自然・宇宙 (70)

### 2. 多言語UIおよび部屋作成ロジック
- **UI切り替え**: `LanguageContext` を通じて、ボタンやラベルなどの全UIテキストを即座に翻訳。
- **RTL対応**: アラビア語選択時に `dir="rtl"` を適用し、レイアウトを反転。
- **部屋の言語設定**: 部屋作成（Lobby）時のUI言語を、Firebase上のルームプロパティ `language` として保存し、ゲーム中に使用される単語リストを決定。

## 実装上の工夫（データ整合性保護）
巨大化した `topics.ts`（約144KB）を安全に編集するため、以下の手法を採用しました：
- **一時ファイル化**: 各言語の生データを `.txt` ファイルとして書き出し。
- **Pythonスクリプトによるマージ**: 通常のテキスト置換ツールでは発生しやすいエンコーディングエラーや部分一致の誤爆を避けるため、正規表現を用いた確実なスクリプト処理を実行。

## 検証計画

- 各言語で部屋を作成し、お題の選択肢がその言語で表示されるか確認。
- アラビア語での表示崩れ（RTL）がないかを確認。
