# 「Just One」機能追加・仕様変更の実装計画

以下の機能追加および仕様変更を実装します。

1.  **ヒントの書き直し機能**: 送信後でもPhase 2の間であれば書き直せるようにします。
2.  **プレイヤー追放（キック）機能**: ホストがプレイヤーを追放できるようにします。
3.  **スコアリングの変更**: 正解で+1、不正解で-1、パスで0点に変更します。
4.  **終了画面の履歴表示**: 各ラウンドのお題、回答、ヒントを履歴として表示します。
5.  **ラウンド数選択機能**: 部屋作成時にラウンド数を指定可能にします。
6.  **フリーモードの追加**: 無限ラウンド、途中参加・離脱可能、ホストがいつでも終了可能なモードです。

## 1. データ構造の変更 (`src/types/game.ts`)

- `Room` 型に `history: Round[]` を追加します。
- `Room` 型に `isFreeMode: boolean` を追加します。
- `Room` 型に `totalRounds` の初期値を動的に設定できるようにします。

## 2. 言語データの追加 (`src/i18n/locales/ja.ts` など)

- フリーモード、ラウンド数選択、キック、履歴表示、書き直しに関する文言を追加します。

## 3. Firebaseロジックの変更 (`src/store/gameStore.ts`)

- `createRoomInFirebase`: `totalRounds` と `isFreeMode` を引数に取るように拡張。
- `finalizeResultInFirebase`: スコア計算ロジックを変更（不正解時に -1）。
- `goNextRoundInFirebase`: `currentRound` を `history` 配列にプッシュするように変更。
- `joinRoomInFirebase`: フリーモード時は待機中以外でも参加を許可。
- **新規関数**:
  - `kickPlayerInFirebase`: 指定したプレイヤーを削除。
  - `undoSubmitHintInFirebase`: 投稿したヒントを削除（書き直し用）。
  - `endFreeModeInFirebase`: フリーモードを強制終了しサマリー画面へ。
  - `leaveRoomInFirebase`: プレイヤー自身が退出する処理。

## 4. UI/コンポーネントの変更

### Lobby.tsx
- 部屋作成時にラウンド数（5, 13, 20など）と「フリーモード」を選択できるUIを追加。

### WaitingRoom.tsx
- ホストの場合、プレイヤー名の横にキックボタンを表示。
- 自身が削除された場合（`room.players[playerId]` が存在しない場合）、ロビーへ自動遷移。

### Game.tsx
- フッターのプレイヤーリストにキックボタンを追加（ホストのみ）。
- フリーモード中、ヘッダーまたはフッターに「ゲームを終了する」ボタンを表示（ホストのみ）。

### Phase2Hint.tsx
- 送信完了画面に「書き直す」ボタンを追加。押すと入力画面に戻る。

### Phase5Result.tsx
- フリーモード中、「次のラウンド」の他に「ゲームを終了する」ボタンを表示。

### GameSummary.tsx
- `room.history` をループして、各ラウンドの記録（お題、正解/不正解、全員のヒント）をリスト表示。

## 5. 検証プラン

- **スコア**: 不正解時にスコアが1減るか確認（0未満になるかも確認）。
- **キック**: 別ブラウザで参加し、ホストからキックされた際にロビーに飛ばされるか確認。
- **履歴**: 複数ラウンドプレイ後、終了画面に全ての履歴が出るか確認。
- **フリーモード**: 途中参加ができるか、ホストがいつでも終わらせられるか確認。
